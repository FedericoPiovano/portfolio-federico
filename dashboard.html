Metodologia del Progetto

1

Data Engineering

Pulizia, gap-filling, feature extraction (lags, medie mobili, variabili calendario, macro-categorie simulate "Agentic").

2

Modello LightGBM

Split temporale (train / val / test). Baseline Seasonal Naive battuta. Bias correction calibrata su validation set.

3

Shadow Mode

Simulazione economica su 3 scenari (Oracolo, LightGBM, Baseline) con prezzi reali per-riga dal dataset.

4

Next Steps

Deploy serverless (AWS Lambda), retraining batch settimanale, integrazione dati LLM per feature testuali avanzate.
